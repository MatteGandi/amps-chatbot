# 6.2 Exercises – Multiple-Product Systems

This section contains **fully worked exercises** on **multiple-product factory models**, applying the methodology introduced in Section 6.1.

The exercises show how the multi-product concept applies to:
- serial lines,
- single workstations,
- networks with feedback.

---

## Exercise 1 – Two Products on a Serial Line

### System Description
- Three workstations in series (WS1 → WS2 → WS3)
- Two product types (Type 1 and Type 2)
- All inter-arrival and processing times are **exponentially distributed**
```
C_A² = C_S² = 1
```

External arrivals:
```
γ₁ = 3 jobs/hour
γ₂ = 1 job/hour
```

---

### Processing Times

| WS | E[TS₁] (h) | E[TS₂] (h) |
|----|------------|------------|
| 1  | 0.20 | 0.20 |
| 2  | 0.15 | 0.15 |
| 3  | 0.14 | 0.25 |

---

## Step 1 – Arrival Rates
Since the system is a serial line:
```
λ₁ = λ₂ = λ₃ = γ₁ + γ₂ = 4 jobs/hour
```

---

## Step 2 – Workload and Utilization

### WS1
```
U₁ = 4 · 0.20 = 0.80
```

### WS2
```
U₂ = 4 · 0.15 = 0.60
```

### WS3
```
U₃ = 3·0.14 + 1·0.25 = 0.67
```

---

## Step 3 – Processing Parameters (Mixture)

### Average Processing Time at WS3
```
E[TS₃] = 0.67 / 4 = 0.1675 hours
```

### Processing Time Variability at WS3
```
C_S3² =
[ (3/4)·0.14²·(1+1) + (1/4)·0.25²·(1+1) ] / 0.1675² − 1
≈ 1.16
```

For WS1 and WS2:
```
C_S1² = C_S2² = 1
```

---

## Step 4 – Arrival Variability
No splitting or merging occurs:
```
C_A1² = C_A2² = C_A3² = 1
```

---

## Step 5 – Performance of Each Workstation

### WS1 (M/M/1)
```
CT₁ = (1+1)/2 · 0.8/(1−0.8) · 0.20 + 0.20 = 1.0 h
WIP₁ = 4 · 1.0 = 4 jobs
```

### WS2 (M/M/1)
```
CT₂ = (1+1)/2 · 0.6/(1−0.6) · 0.15 + 0.15 = 0.375 h
WIP₂ = 4 · 0.375 = 1.5 jobs
```

### WS3 (G/G/1)
```
CT₃ ≈ (1+1.16)/2 · 0.67/(1−0.67) · 0.1675 + 0.1675 ≈ 0.76 h
WIP₃ = 4 · 0.76 ≈ 3.04 jobs
```

---

## System Performance
```
WIP_total = 4 + 1.5 + 3.04 ≈ 8.54 jobs
CT_total = WIP_total / 4 ≈ 2.14 hours
```

---

## Cycle Time by Product Type

For product *i*:
```
CT_i = Σ_k CT_{ik}
```

Average total cycle time:
```
CT = (3/4)·CT₁ + (1/4)·CT₂
```

---

## Exercise 2 – Network with Rework

### Description
- Three workstations
- Two product types
- Product 1 is reworked at WS3 with probability 0.1

External arrivals:
```
γ₁ = 2 jobs/hour
γ₂ = 1 job/hour
```

Processing times (exponential):

| WS | E[TS₁] | E[TS₂] |
|----|--------|--------|
| 1  | 0.30 | – |
| 2  | – | 0.40 |
| 3  | 0.10 | 0.20 |

---

## Step 1 – Arrival Rates

Rework flow:
```
0.1 · γ₁ = 0.2 jobs/hour
```

Arrival rates:
```
λ₁ = 2
λ₂ = 1
λ₃ = 2 + 1 + 0.2 = 3.2 jobs/hour
```

---

## Step 2 – Utilization

### WS1
```
U₁ = 2 · 0.30 = 0.60
```

### WS2
```
U₂ = 1 · 0.40 = 0.40
```

### WS3
```
U₃ = 2·0.10 + 1·0.20 + 0.2·0.10 = 0.42
```

---

## Step 3 – Processing Parameters at WS3

Average processing time:
```
E[TS₃] = 0.42 / 3.2 ≈ 0.13 h
```

Processing time variability:
```
C_S3² ≈ 1.23
```

---

## Step 4 – Arrival Variability at WS3

Using merging + feedback:
```
C_A3² ≈ 1.00
```

---

## Step 5 – Performance

### WS1
```
CT₁ = 0.75 h
WIP₁ = 1.5
```

### WS2
```
CT₂ = 0.67 h
WIP₂ = 0.67
```

### WS3
```
CT₃ = 0.24 h
WIP₃ = 0.76
```

---

## System Results
```
WIP_total ≈ 2.93 jobs
TH = γ₁ + γ₂ = 3 jobs/hour
CT_total = 2.93 / 3 ≈ 0.98 hours
```

---

## Key Takeaways
- Multi-product systems require workload aggregation.
- Processing times are mixtures of distributions.
- Rework introduces feedback and variability.
- The general network approximation model applies.
