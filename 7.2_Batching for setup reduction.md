# 7.2 Batching for Setup Reduction

## 7.2.1 Motivation
The second form of batching is **batching for setup reduction**.

Unlike transport batching:
- items are still processed **individually**,
- but a **setup time** is incurred **once per batch**, not per item.

Batching is therefore used to **amortize setup times** over multiple items.

---

## 7.2.2 System Structure

The system operates as follows:

1. Individual items arrive
2. Items are grouped into batches of size **k**
3. Batches wait in queue at the workstation
4. A **setup** is performed once per batch
5. Items in the batch are processed one by one

Transport time is neglected, but:
- batch forming time,
- batch queueing time,
- setup time,
- processing time  
all contribute to cycle time.

---

## 7.2.3 Input Parameters

### Individual Item Parameters
- Average processing time:
```
E[TS_i]
```
- Processing-time variability:
```
C_S,i²
```

### Setup Parameters
- Average setup time:
```
E[R]
```
- Setup-time variability:
```
C_R²
```

### Arrival Parameters
- Arrival rate of individuals:
```
λ_i
```
- Arrival variability:
```
C_A,i²
```

### Batch Size
```
k
```

---

## 7.2.4 Processing Time of a Batch

The processing time of a batch is:
```
TS_batch = R + Σ_{j=1}^k TS_i,j
```

### Mean Processing Time
```
E[TS_batch] = E[R] + k · E[TS_i]
```

---

### Variability of Batch Processing Time
The variance is:
```
Var(TS_batch) = Var(R) + k · Var(TS_i)
```

Using squared coefficients of variation:
```
C_S,batch² =
[ C_R² · E[R]² + k · C_S,i² · E[TS_i]² ]
/ (E[R] + k·E[TS_i])²
```

---

## 7.2.5 Utilization

Utilization is computed **per individual item**, not per batch.

Average processing time per item:
```
E[TS_eff] = E[R]/k + E[TS_i]
```

Utilization:
```
U = λ_i · (E[R]/k + E[TS_i])
```

The utilization depends on **k**.

---

## 7.2.6 Stability Condition
To avoid infinite queues:
```
U < 1
```

This condition determines the **minimum admissible batch size**.

---

## 7.2.7 Batch Forming Time
As in transport batching, items wait to form a batch.

Average batch forming time:
```
CT_form = (k − 1)/2 · E[T_A,i]
```

---

## 7.2.8 Queueing Time of Batches

Batches queue at the workstation.

Arrival variability of batches:
```
C_A,batch² = C_A,i² / k
```

Queue cycle time (VUT on batches):
```
CTQ_batch ≈ (C_A,batch² + C_S,batch²)/2 · U/(1 − U) · E[TS_batch]
```

---

## 7.2.9 Setup and Processing Delay

All items in the batch wait for:
- the setup time
- the processing of previous items

Average internal delay:
```
CT_delay = E[R] + (k − 1)/2 · E[TS_i]
```

---

## 7.2.10 Total Cycle Time (Individual Item)

The average cycle time of an item is:

```
CT =
(k − 1)/2 · E[T_A,i]
+ (C_A,i²/k + C_S,batch²)/2 · U/(1 − U) · (E[R] + k·E[TS_i])
+ E[R]
+ (k − 1)/2 · E[TS_i]
+ E[TS_i]
```

This is the **reference formula** for batching for setup reduction.

---

## 7.2.11 Worked Example – Optimal Batch Size

### Given
- Individual processing time:
```
E[TS_i] = 0.1 h
C_S,i² = 1.5
```
- Setup time:
```
E[R] = 0.2 h
C_R² = 1
```
- Arrival process:
```
λ_i = 5.66 items/h
E[T_A,i] = 0.1765 h
C_A,i² = 3
```

---

## Step 1 – Utilization Constraint

Utilization:
```
U = 5.66 · (0.2/k + 0.1)
```

Stability condition:
```
U < 1  →  k ≥ 3
```

Admissible batch sizes:
```
k = 3, 4, 5, ...
```

---

## Step 2 – Cycle Time Evaluation

Cycle time is computed for increasing values of **k** using the formula above.

Results:

| Batch size k | Cycle time (h) |
|--------------|----------------|
| 3 | 6.24 |
| 4 | 2.40 |
| 5 | 2.02 |
| 6 | 1.86 |
| 7 | ↑ |
| 8 | ↑ |

---

## Step 3 – Optimal Batch Size

The cycle time:
- decreases initially as setup time is amortized,
- increases for large k due to batch waiting and internal delays.

**Optimal batch size**:
```
k* = 6
```

---

## 7.2.12 Key Takeaways
- Batching for setup reduction amortizes setup times.
- Utilization depends on batch size.
- Small k → setup dominates.
- Large k → waiting dominates.
- An optimal batch size minimizing cycle time exists.
