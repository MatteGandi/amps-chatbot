# 4.2 Machine Breakdown and Processing Time Variability

## 4.2.1 Role of Machine Breakdowns
Machine breakdowns are a **second major source of processing time variability**, in addition to natural variability.

When a breakdown occurs:
- the machine stops,
- a **repair time** is required,
- the completion of the current job is delayed.

In this chapter, we assume:
- the job under processing is **not damaged**,
- after repair, processing resumes normally,
- no scrapping or rework is explicitly modeled.

Thus, breakdowns affect **time only**, not quality.

---

## 4.2.2 Natural vs Effective Processing Time

We distinguish between:
- **Natural processing time (TS)**: processing time without failures
- **Effective processing time (TE)**: processing time including breakdowns and repairs

Breakdowns introduce **additional time** due to repairs, increasing both:
- average processing time,
- processing time variability.

---

## 4.2.3 Machine Availability

Machine availability is defined as:
```
A = E[F] / (E[F] + E[R])
```

Where:
- E[F] = average time between failures
- E[R] = average repair time

Properties:
- 0 < A ≤ 1
- A = 1 → perfectly reliable machine

---

## 4.2.4 Average Effective Processing Time

The expected effective processing time is:
```
E[TE] = E[TS] / A
```

Since A ≤ 1:
```
E[TE] ≥ E[TS]
```

Breakdowns **always increase** the average processing time.

---

## 4.2.5 Effective Variability of Processing Time

Breakdowns also increase variability.
The **effective squared coefficient of variation** is:

```
C_E² = C_S² + (1 + C_R²) · A · (1 − A) · (E[R] / E[TS])
```

Where:
- C_S² = natural processing time variability
- C_R² = repair time variability
- A = availability

This formula is **given** (not derived) and will be provided in the exam.

---

## 4.2.6 How to Use Breakdown Data in Queueing Models

When breakdowns are present:
1. Compute availability A
2. Compute E[TE]
3. Compute C_E²
4. Use **E[TE]** and **C_E²** in the VUT / Kingman formula

The general queue cycle time formula becomes:
```
CTQ ≈ (C_A² + C_E²)/2 · U/(1 − U) · E[TE]
```

If no breakdowns occur:
```
C_E² = C_S²
E[TE] = E[TS]
```

---

## 4.2.7 Example 1 – Single Workstation with Breakdown

### Input Data
- Average inter-arrival time: E[TA] = 75 min
- Natural processing time: E[TS] = 58 min
- Arrival process: exponential → C_A² = 1
- Processing time: Erlang-3 → C_S² = 1/3

### Performance Without Breakdowns
Utilization:
```
U = E[TS] / E[TA] = 0.773
```

Queue cycle time (VUT):
```
CTQ ≈ 132 min
```

---

### Breakdown Data
- Mean time between failures: E[F] = 3 h
- Mean repair time: E[R] = 30 min
- Repair time std dev: σ_R = 15 min

Repair time variability:
```
C_R² = (15²) / (30²) = 0.25
```

Availability:
```
A = 3 / (3 + 0.5) ≈ 0.857
```

---

### Effective Parameters
Effective processing time:
```
E[TE] = 58 / 0.857 ≈ 67.7 min
```

New utilization:
```
U = 67.7 / 75 ≈ 0.902
```

Effective variability:
```
C_E² ≈ 0.413
```

---

### Performance With Breakdowns
Queue cycle time:
```
CTQ ≈ 441 min
```

**Conclusion:** breakdowns dramatically worsen performance, especially at high utilization.

---

## 4.2.8 Key Insight on Utilization
The closer utilization is to 1:
- the stronger the impact of breakdowns,
- the larger the increase in waiting time and WIP.

Ignoring breakdowns leads to **severely underestimated** performance measures.

---

## 4.2.9 Example 2 – Choosing Between Two Machines

Two machines have identical natural processing times:
- E[TS] = 15 min
- σ_TS = 3.35 min

Both have the **same availability** A = 0.75, but different breakdown patterns.

### Machine M1
- E[F] = 744 min
- E[R] = 248 min
- Rare but long failures

### Machine M2
- E[F] = 140 min
- E[R] = 38 min
- Frequent but short failures

---

### Comparison
- Availability: identical
- E[TE]: identical
- Utilization: identical

Difference lies in **variability**:

- M1 → high C_E² (high variability)
- M2 → moderate C_E²

---

## 4.2.10 Decision Rule
When choosing between machines:
- **lower effective variability is preferable**,
- frequent short failures are better than rare long failures,
- variability dominates average values in queueing performance.

---

## 4.2.11 Key Takeaways
- Breakdowns increase both average processing time and variability.
- Availability affects utilization and performance.
- Effective parameters must be computed **before** applying queueing formulas.
- Variability, not just availability, determines system performance.
