# 4.3 Exercises – Machine Breakdowns and Queueing Performance

This section collects and **solves step by step** the exercises related to **machine breakdowns**, **effective processing time**, and **queueing performance**.

The exercises must be addressed **in order**:
1. Exercise 1  
2. Exercise 3  
3. Exercise 2 (most complex – limited system)

---

## Exercise 1 – Effective Processing Time Parameters

### Given
- Natural processing time:
```
E[TS] = 3 hours
C_S² = 2
```
- Time between failures:
```
E[F] = 7 hours
C_F² = 1
```
- Repair time:
```
E[R] = 1 hour
C_R² = 1
```

All distributions are exponential unless otherwise specified.

---

### Step 1 – Availability
```
A = E[F] / (E[F] + E[R]) = 7 / (7 + 1) = 0.875
```

---

### Step 2 – Effective Processing Time
```
E[TE] = E[TS] / A = 3 / 0.875 ≈ 3.43 hours
```

---

### Step 3 – Effective Variability
```
C_E² = C_S² + (1 + C_R²) · A · (1 − A) · (E[R] / E[TS])
```
```
C_E² = 2 + (1 + 1) · 0.875 · 0.125 · (1 / 3) ≈ 2.073
```

---

### Step 4 – Variance of Effective Processing Time
```
Var(TE) = C_E² · E[TE]² ≈ 24.39 hours²
```

## Exercise 3 – M/M/1 with Breakdowns (Unlimited System)

### Given
- Arrival rate:
```
λ = 5 jobs/hour
```
- Service rate:
```
μ = 7 jobs/hour
```
- Breakdown rate:
```
α = 1 / hour
```
- Mean repair time:
```
E[R] = 15 minutes = 0.25 hours
```
- All distributions exponential:
```
C_A² = C_S² = C_R² = 1
```

---

### Step 1 – Availability
```
A = 1 / (1 + 0.25) = 0.8
```

---

### Step 2 – Effective Processing Time
```
E[TS] = 1 / μ = 0.143 hours
E[TE] = E[TS] / A ≈ 0.18 hours
```

---

### Step 3 – Effective Variability
```
C_E² = 1 + (1 + 1) · 0.8 · 0.2 · (0.25 / 0.143) ≈ 1.56
```

---

### Step 4 – Utilization
```
U = λ · E[TE] = 5 · 0.18 ≈ 0.9
```

---

### Step 5 – Queue Cycle Time (VUT)
```
CTQ ≈ (C_A² + C_E²)/2 · U/(1 − U) · E[TE]
CTQ ≈ 2.07 hours
```

---

### Step 6 – Total Cycle Time
```
CT = CTQ + E[TE] ≈ 2.25 hours
```

---

### Step 7 – Throughput and WIP
```
TH = λ = 5 jobs/hour
WIP = TH · CT ≈ 11.25 jobs
```

---

## Exercise 2 – M/M/1/3 System with Breakdowns (Limited System)

### Given
- Arrival rate:
```
λ = 5 jobs/hour
```
- Service rate:
```
μ = 6 jobs/hour
```
- Breakdown rate:
```
α = 1 / hour
```
- Repair rate:
```
β = 4 / hour
```
- Maximum WIP:
```
K = 3
```

---

## State Description
Each state is defined by:
- number of jobs in the system
- machine condition: Processing (P) or Breakdown (B)

States:
```
0
1P, 1B
2P, 2B
3P, 3B
```

Arrivals are blocked in states **3P** and **3B**.

---

## Balance Equations (Structure)
```
λP0 = μP1P
(μ + λ + α)P1P = λP0 + μP2P + βP1B
(β + λ)P1B = αP1P
...
Σ Pi = 1
```

The system is solved numerically.

---

## Steady-State Probabilities (Results)
```
P0  = 0.138
P1P = 0.061
P1B = 0.056
P2P = 0.236
P2B = 0.028
P3P = 0.330
P3B = 0.079
```

---

## Machine States
- Idle:
```
P_idle = P0 = 0.138
```
- Processing:
```
P_proc = P1P + P2P + P3P = 0.739
```
- Breakdown:
```
P_down = P1B + P2B + P3B = 0.121
```

---

## Performance Measures

### Work In Process
```
WIP = 1(P1P + P1B) + 2(P2P + P2B) + 3(P3P + P3B)
WIP ≈ 1.94 jobs
```

---

### Queue WIP
```
WIP_Q = 1(P2P + P2B) + 2(P3P + P3B)
WIP_Q ≈ 1.08 jobs
```

---

### Throughput
```
TH = λ · (1 − P3P − P3B) ≈ 2.96 jobs/hour
```

---

### Cycle Time
```
CT = WIP / TH ≈ 0.66 hours
```

---

## Key Takeaways
- Effective parameters must be computed **before** queueing formulas.
- Breakdowns strongly affect performance at high utilization.
- Limited systems require **state-based analysis**.
- Throughput is reduced by blocking.
