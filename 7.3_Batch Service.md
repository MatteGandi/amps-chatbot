# 7.3 Batch Service (Batch Machines)

## 7.3.1 Introduction

After:
- batching for transport
- batching for setup reduction

we now consider the third case:

> **Batch service**, where the entire batch is processed simultaneously.

Typical example:
- ovens
- furnaces
- heat-treatment chambers
- washing machines

Items arrive individually, batches are formed, and the **whole batch is processed at once**.

---

## 7.3.2 System Structure

1. Items arrive one at a time
2. A batch of size **k** is formed
3. Batches wait in queue
4. The workstation processes the **entire batch simultaneously**

Key difference from setup batching:
- processing time refers to the **whole batch**, not per item.

---

## 7.3.3 Input Parameters

### Individual Arrival Parameters
- Arrival rate:
```
λ_i
```
- Squared coefficient of variation:
```
C_A,i²
```

### Batch Parameters
- Batch size:
```
k
```
- Mean batch processing time:
```
E[T_S,batch]
```
- Processing variability:
```
C_S,batch²
```

---

## 7.3.4 Cycle Time Derivation

Three components contribute to cycle time of an individual item:

### 1. Batch Forming Time
If items arrive individually:
```
CT_form = (k − 1)/2 · E[T_A,i]
```

If batches arrive already formed:
```
CT_form = 0
```

---

### 2. Waiting Time in Queue (Batch Level)

Arrival variability of batches:
```
C_A,batch² = C_A,i² / k
```

Queue cycle time (VUT applied to batches):
```
CTQ_batch ≈ (C_A,batch² + C_S,batch²)/2 · U/(1 − U) · E[T_S,batch]
```

---

### 3. Processing Time

Since the entire batch is processed simultaneously:

Each item experiences the same processing time:
```
E[T_S,batch]
```

---

## 7.3.5 Total Cycle Time (Individual Item)

If items arrive individually:

```
CT =
(k − 1)/2 · E[T_A,i]
+ (C_A,i²/k + C_S,batch²)/2 · U/(1 − U) · E[T_S,batch]
+ E[T_S,batch]
```

If batches arrive already formed:

```
CT =
(C_A,batch² + C_S,batch²)/2 · U/(1 − U) · E[T_S,batch]
+ E[T_S,batch]
```

---

## 7.3.6 Departure Variability

### Departure Variability of Batches

Use standard departure formula:

```
C_D,batch² ≈ (1 − U²)·C_A,batch² + U²·C_S,batch²
```

---

### Departure Variability of Individual Items

If after processing items leave individually:

```
C_D,item² = k · C_D,batch² + (k − 1)
```

If batches move downstream:

```
C_D,next = C_D,batch²
```

This distinction is essential for network analysis.

---

## 7.3.7 Worked Example

### Given
- Batch size:
```
k = 5
```
- Arrival rate of batches:
```
λ_batch = 3 batches/hour
```
- Arrival variability:
```
C_A,batch² = 0.75
```
- Processing variability:
```
C_S,batch² = 0.75
```
- Utilization:
```
U = 0.84
```

---

### Step 1 – Processing Time of Batch

From utilization:

```
U = λ_batch · E[T_S,batch]
```

```
E[T_S,batch] = U / λ_batch = 0.84 / 3 = 0.28 hours
```

```
= 16.8 minutes
```

---

### Step 2 – Cycle Time

No forming time (batches already formed):

```
CT =
(0.75 + 0.75)/2 · 0.84/(1 − 0.84) · 0.28
+ 0.28
```

```
CT ≈ 1.95 hours
```

---

### Step 3 – Departure Variability (Batch)

Using departure formula:

```
C_D,batch² = 0.75
```

---

### Step 4 – Departure Variability (Individual Items)

```
C_D,item² = 5 · 0.75 + 4 = 7.75
```

This value must be used if items move individually downstream.

---

## 7.3.8 Key Takeaways

- Batch service processes entire batches simultaneously.
- Cycle time includes:
  - forming time (if applicable)
  - queueing time (batch level)
  - batch processing time
- Departure variability depends on:
  - whether flow remains in batches
  - or splits into individual items
- Batch machines can dramatically increase variability downstream.
