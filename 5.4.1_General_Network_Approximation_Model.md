# 5.4.1 General Network Approximation Model

## 5.4.1.1 Motivation
In serial systems, workstations can be analyzed **one at a time**.
In **general networks**, this is no longer possible because:
- flows may re-enter upstream workstations,
- arrival rates depend on departures of other stations,
- stations are **mutually dependent**.

Therefore, arrival parameters must be computed by solving a **system of equations**.

---

## 5.4.1.2 External Arrivals and Notation
- External arrival rate at workstation *i*:  
```
γ_i
```
- Total arrival rate at workstation *i*:  
```
λ_i
```

External arrivals are **not included** in the routing matrix.

---

## 5.4.1.3 Routing Matrix

### Definition
The **routing matrix** `P` collects all routing probabilities between workstations.

Element:
```
p_{ij} = probability that a job leaving workstation j goes to workstation i
```

Properties:
- Rows → **splitting**
- Columns → **merging**
- External exits are not represented

---

### Example (Two-Workstation Network)
Routing matrix:
```
P = | 0      1 |
    | β   α |
```

---

## 5.4.1.4 Arrival Rate Equations

For a generic workstation *i*:
```
λ_i = γ_i + Σ_{k=1}^n p_{ki} · λ_k
```

This produces a **system of linear equations**.

---

### Matrix Form
Let:
- `λ` = vector of arrival rates
- `γ` = vector of external arrivals

Then:
```
λ = Pᵀ · λ + γ
```

Rearranging:
```
(I − Pᵀ) · λ = γ
```

Solution:
```
λ = (I − Pᵀ)⁻¹ · γ
```

In exams, systems are small and can be solved by **substitution**.

---

## 5.4.1.5 Example – Network with Feedback

Given:
- External arrival to WS1:
```
γ₁ = 5 jobs/hour
```
- Routing probabilities:
```
p_{21} = 1/10
p_{31} = 1/20
p_{12} = 3/4
p_{13} = 1/4
p_{23} = 9/10
```

Arrival-rate system:
```
λ₁ = 5 + (1/10)λ₂ + (1/20)λ₃
λ₂ = (3/4)λ₁
λ₃ = (1/4)λ₁ + (9/10)λ₂
```

Solving:
```
λ₁ ≈ 5.69
λ₂ ≈ 4.27
λ₃ ≈ 5.26   jobs/hour
```

---

## 5.4.1.6 Consistency Check
Always verify:
```
Total external inflow = total network outflow
```

This check ensures the system has been solved correctly.

---

## 5.4.1.7 Arrival Variability in Networks

Arrival variability at workstation *j* is computed using a **merging rule**.

---

### General Formula
```
C_{A,j}² = (γ_j / λ_j) · C_{A,0j}²
          + Σ_{k=1}^n (p_{kj}·λ_k / λ_j) · C_{A,kj}²
```

Where:
- `C_{A,0j}²` = variability of external arrivals
- `C_{A,kj}²` = variability of flow from k → j

---

## 5.4.1.8 Variability of Routed Flows

For a routed flow:
```
C_{A,kj}² = p_{kj}·C_{D,k}² + (1 − p_{kj})
```

Departure variability:
```
C_{D,k}² ≈ (1 − U_k²)·C_{A,k}² + U_k²·C_{S,k}²
```

Where:
- `U_k` = utilization at workstation *k*
- `C_{S,k}²` = processing-time variability

---

## 5.4.1.9 Full Procedure for Networks

1. Build routing matrix `P`
2. Solve `(I − Pᵀ)·λ = γ`
3. Compute utilizations `U_i`
4. Compute departure variabilities `C_{D,i}²`
5. Solve variability system for `C_{A,i}²`
6. Use VUT / Kingman for performance evaluation

---

## 5.4.1.10 Key Takeaways
- General networks require **system-level analysis**
- Routing matrix encodes splitting and merging
- Arrival rates and variabilities are interdependent
- WIP is additive; cycle time is obtained via Little’s Law
- This model generalizes **all previous cases**
