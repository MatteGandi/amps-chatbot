# 7.4 Batch Line Exercise – Complete Worked Example

This section provides a **full worked example** of a **production line with batching**, combining:
- parallel machines,
- machine breakdowns,
- transport batching,
- downstream variability propagation.

The exercise integrates concepts from Chapters **3, 4, 5, and 7**.

---

## System Overview

The production system consists of **four workstations in series**:

1. **WS1** – Three parallel machines  
2. **WS2** – Single machine with breakdowns  
3. **WS3** – Transport batching (batch size k = 40)  
4. **WS4** – Single downstream workstation  

External arrivals:
```
λ = 5 jobs/hour
```

---

## Workstation 1 – Parallel Machines (G/G/3)

### Processing Time
- Mean:
```
E[TS₁] = 0.5 hours
```
- Standard deviation:
```
σ₁ = 15 minutes = 0.25 hours
```
- Variability:
```
C_S1² = (0.25 / 0.5)² = 0.25
```

### Utilization
```
U₁ = λ · E[TS₁] / 3 = 5 · 0.5 / 3 ≈ 0.83
```

### Cycle Time
Using the approximation for **G/G/c** systems:
```
CT₁ ≈ 1.63 hours
```

### Departure Variability
```
C_D1² ≈ 0.70
```

---

## Workstation 2 – Machine with Breakdowns

### Natural Processing Time
```
E[TS₂] = 8 minutes = 0.133 hours
C_S2² = 1
```

### Breakdown Data
- Mean time between failures:
```
E[F] = 60 minutes = 1 hour
```
- Mean repair time:
```
E[R] = 8 minutes = 0.133 hours
```
- Repair time variability:
```
C_R² = 1
```

### Availability
```
A = E[F] / (E[F] + E[R]) = 1 / (1 + 0.133) ≈ 0.88
```

### Effective Processing Time
```
E[TE₂] = E[TS₂] / A ≈ 0.15 hours
```

### Effective Variability
```
C_E2² ≈ 1.13
```

### Utilization
```
U₂ = λ · E[TE₂] = 5 · 0.15 = 0.75
```

### Cycle Time
```
CT₂ ≈ 0.57 hours
```

### Departure Variability
```
C_D2² ≈ 0.94
```

---

## Transport Batching After WS2

### Batch Size
```
k = 40
```

### Batch Forming Time
```
CT_form = (k − 1)/2 · (1 / λ) = 39/2 · 0.2 ≈ 3.9 hours
```

---

## Workstation 3 – Processing After Transport Batching

### Processing Time
```
E[TS₃] = 8 minutes = 0.133 hours
C_S3² = 1
```

### Utilization
```
U₃ = λ · E[TS₃] = 5 · 0.133 ≈ 0.67
```

### Cycle Time (Batching for Transport)
```
CT₃ ≈ 6.89 hours
```

### Departure Variability
```
C_D3² ≈ 5.30
```

---

## Workstation 4 – Downstream Single Machine

### Processing Time
```
E[TS₄] = 10 minutes = 0.167 hours
C_S4² = 1
```

### Utilization
```
U₄ = λ · E[TS₄] = 5 · 0.167 ≈ 0.83
```

### Arrival Variability
```
C_A4² = C_D3² ≈ 5.30
```

### Cycle Time
```
CT₄ ≈ 2.79 hours
```

---

## Total System Performance

### Total Cycle Time
Since the system is a **serial line**:
```
CT_total = CT₁ + CT₂ + CT₃ + CT₄
```

```
CT_total ≈ 11.2 hours
```

Transport time is **not included**.  
If transport time is known, it can be added directly.

---

## Key Takeaways

- Batch forming time can dominate total cycle time.
- Batching does **not** change utilization but greatly increases CT.
- Variability explodes downstream after batching.
- All effects must be modeled explicitly.
- This exercise integrates **parallel machines, breakdowns, batching, and flow variability**.
