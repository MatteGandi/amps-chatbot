# 4.1 Introduction to Variability of Processing Time
## Natural Processing Time Variability

## 4.1.1 Connection with the VUT (Kingman) Equation
We recall the queue cycle time approximation for a **G/G/1** system:

```
CTQ ≈ (C_A² + C_S²)/2 · U/(1 − U) · E[TS]
```

Where:
- C_A² = squared coefficient of variation of inter-arrival times
- C_S² = squared coefficient of variation of processing times
- U = utilization
- E[TS] = average processing time

The objective is to **minimize CTQ**, i.e. the average waiting time in queue.

---

## 4.1.2 Levers for Reducing Queue Cycle Time
From the VUT equation, CTQ can be reduced by:
1. Reducing utilization U
2. Reducing variability (C_A² or C_S²)

Reducing variability has an effect equivalent to reducing utilization, but **without reducing throughput**.

---

## 4.1.3 Numerical Example: Variability vs Utilization

Given:
- C_A² = 1
- C_S² = 1
- U = 0.8
- E[TS] = 2 hours

Queue cycle time:
```
CTQ = (1 + 1)/2 · 0.8/(1 − 0.8) · 2 = 8 hours
```

### Reducing Processing Time Variability
If:
- C_S² = 0.9
- other parameters unchanged

Then:
```
CTQ = 7.6 hours
```

### Equivalent Effect via Utilization Reduction
Keeping:
- C_S² = 1

To obtain:
```
CTQ = 7.6 hours
```

Required utilization:
```
U ≈ 0.792
```

**Conclusion:**  
Reducing variability is equivalent to reducing utilization, but superior because capacity is preserved.

---

## 4.1.4 Graphical Interpretation
- As utilization increases, CTQ increases sharply.
- Reducing variability shifts the CTQ–U curve downward.
- This allows high utilization with lower waiting times.

---

## 4.1.5 Sources of Processing Time Variability
Main contributors:
1. Natural (intrinsic) variability
2. Breakdowns and repairs
3. Operator-related variability
4. Setups (batch-related)

This section focuses on **natural processing time variability**.

---

## 4.1.6 Definition of Natural Processing Time Variability
Natural variability is the intrinsic randomness of processing times:
- processing times differ across jobs,
- even under stable operating conditions.

This variability is captured by **C_S²**.

---

## 4.1.7 Decomposition of Processing Time

Let the total processing time be:
```
T = T₁ + T₂ + ... + Tₖ
```

Assume subprocesses are **independent and identically distributed (i.i.d.)**.

### Mean
```
E[T] = k · E[Tᵢ]
```

### Variance
```
Var(T) = k · Var(Tᵢ)
```

### Coefficient of Variation
```
C_T² = Var(T) / E[T]² = (1/k) · C_Tᵢ²
```

**Key result:**  
Increasing the number of subprocesses reduces total variability.

---

## 4.1.8 Numerical Example: Three Subtasks

Initial process:
- E[T] = 3 hours
- C_T² = 1 (exponential)

Three identical subtasks:
- E[Tᵢ] = 1 hour
- Var(Tᵢ) = 3
- C_Tᵢ² = 3

### Variability Reduction at Subtask Level
If subtask variability is reduced to:
```
C_Tᵢ² = 1
```

Then:
```
Var(Tᵢ) = 1
Var(T) = 3
C_T² = 3 / 9 = 1/3
```

Total variability is reduced by a factor of three.

---

## 4.1.9 Effect on System Performance
Reducing variability of subprocesses:
- does not change average processing time,
- significantly reduces total variability,
- reduces CTQ through the VUT equation.

This is the **only way to reduce natural variability**.

## 4.1.11 Key Takeaways
- Natural variability is intrinsic to processes.
- It cannot be eliminated directly.
- Decomposition into subprocesses reduces variability.
- Lower variability leads to lower waiting times.
- Variability reduction is equivalent to utilization reduction, but more effective.
