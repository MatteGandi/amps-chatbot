# 5.4.2 General Network Approximation Model – Worked Example

## 5.4.2.1 Objective
This section provides a **complete worked example** of the **general network approximation model** introduced in Section 5.4.1.

The goal is to compute:
- arrival rates at each workstation,
- utilizations,
- arrival variability,
- individual workstation performance,
- overall network performance.

---

## 5.4.2.2 Summary of the General Procedure
To analyze a general network:

1. Compute **mean arrival rates** at each workstation  
2. Compute **utilizations**  
3. Compute **arrival variability** (solve a system of equations)  
4. Compute **cycle time and WIP** of each workstation  
5. Compute **total WIP** and **total cycle time** using Little’s Law

---

## 5.4.2.3 System Description

### Network Structure
- 4 single-server workstations
- One external arrival stream entering **only WS1**
- Internal flows include splitting, merging, and feedback

External arrivals:
```
γ₁ = 5 jobs/hour
```
External arrivals are exponential:
```
C_A,ext² = 1
```

---

### Processing Parameters

| WS | E[TS] (h) | C_S² |
|----|-----------|------|
| 1  | 0.13 | 1.0355 |
| 2  | 0.13 | 1.7751 |
| 3  | 0.16 | 0.3906 |
| 4  | —    | 2.414  |

---

## 5.4.2.4 Routing Matrix

Routing matrix **P**:

```
      WS1   WS2   WS3   WS4
WS1    0    2/3   1/3    0
WS2    0     0     0     1
WS3    0     0     0     1
WS4   1/10  2/10  3/10   0
```

Rows represent **splitting**, columns represent **merging**.

---

## 5.4.2.5 Arrival Rate Equations

Arrival rates are obtained from:
```
λ = Pᵀ·λ + γ
```

Explicit system:
```
λ₁ = 5 + (1/10)λ₄
λ₂ = (2/3)λ₁ + (2/10)λ₄
λ₃ = (1/3)λ₁ + (3/10)λ₄
λ₄ = λ₂ + λ₃
```

Solution:
```
λ₁ = 6.25
λ₂ = 6.67
λ₃ = 5.83
λ₄ = 12.5     jobs/hour
```

---

## 5.4.2.6 Utilization of Workstations

Utilization formula:
```
U_i = λ_i · E[TS_i]
```

Results:

| WS | U | U² | 1 − U² |
|----|----|----|--------|
| 1 | 0.8125 | 0.6602 | 0.3398 |
| 2 | 0.8667 | 0.7512 | 0.2488 |
| 3 | 0.8710 | 0.7590 | 0.2410 |
| 4 | 0.8000 | 0.6400 | 0.3600 |

---

## 5.4.2.7 Arrival Variability Equations

Arrival variability is computed using the **merging formula**.

### Example – WS1
Two incoming flows:
- external arrivals
- feedback from WS4

Resulting equation:
```
C_A1² = 0.0072·C_A4² + 1.0112
```

---

### WS2, WS3, WS4
Similarly obtained equations:
```
C_A2² = 0.1416·C_A1² + 0.027·C_A4² + 0.91
C_A3² = 0.0405·C_A1² + 0.694·C_A2² + 0.0602
C_A4² = C_A2² + C_A3²
```

---

### Solution of Variability System
Solving the system:
```
C_A1² = 1.0199
C_A2² = 1.0840
C_A3² = 1.1874
C_A4² = 1.0850
```

---

## 5.4.2.8 Performance of Individual Workstations

Cycle time (VUT):
```
CT_i = (C_Ai² + C_Si²)/2 · U_i/(1 − U_i) · E[TS_i] + E[TS_i]
```

### Example – WS1
```
CT₁ ≈ 0.709 hours
WIP₁ = λ₁ · CT₁ ≈ 4.43 jobs
```

The same computation is repeated for WS2, WS3, WS4.

---

## 5.4.2.9 Network Performance

Total WIP:
```
WIP_total = Σ WIP_i ≈ 31.03 jobs
```

Throughput:
```
TH = γ₁ = 5 jobs/hour
```

Total cycle time:
```
CT_total = WIP_total / TH ≈ 6.2 hours
```

---

## 5.4.2.10 Key Takeaways
- General networks require solving **two systems of equations**:
  - arrival rates
  - arrival variability
- Feedback strongly increases variability.
- WIP is additive; cycle time is obtained via Little’s Law.
- This procedure generalizes all previous models.
