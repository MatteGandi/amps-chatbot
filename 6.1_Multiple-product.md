# 6.1 Multiple-Product Factory Models

## 6.1.1 Motivation
Until now, the analysis focused on:
- single workstations,
- serial lines,
- general networks,
assuming **a single product type**.

In reality, most manufacturing systems process **multiple product types**, each characterized by:
- different arrival rates,
- different processing times,
- different routing structures.

This chapter introduces **multiple-product factory models** and shows how to extend previous methods.

---

## 6.1.2 Key Modeling Idea
The presence of multiple product types does **not change the logic** of the analysis.
Instead, it requires:
- additional indices,
- aggregation of parameters across product types,
- careful use of flow conservation by product type.

The system is still analyzed **workstation by workstation**.

---

## 6.1.3 Notation for Multiple Products

### Indices
- *i* = product type (i = 1, …, m)
- *k* = workstation (k = 1, …, n)

---

### Arrival Rates
- Arrival rate of product *i* at workstation *k*:
```
λ_{ik}
```

- External arrival rate of product *i* to workstation *k*:
```
γ_{ik}
```

- Total external arrival rate of product *i*:
```
γ_i = Σ_{k=1}^n γ_{ik}
```

- Total arrival rate at workstation *k*:
```
λ_k = Σ_{i=1}^m λ_{ik}
```

---

## 6.1.4 Workload and Utilization

### Workload at Workstation *k*
```
WL_k = Σ_{i=1}^m λ_{ik} · E[TS_{ik}]
```

- If workstation *k* has **one machine**:
```
U_k = WL_k
```

- If workstation *k* has **c_k parallel machines**:
```
U_k = WL_k / c_k
```

---

## 6.1.5 Average Processing Time (Mixture Distribution)

The processing time at a workstation is a **mixture of random variables**, depending on the product type.

### Average Processing Time
```
E[TS_k] = WL_k / λ_k
```

This is a **weighted average** of processing times across product types.

---

## 6.1.6 Variability of Processing Time

Let:
- `C_{S,ik}²` = squared coefficient of variation of processing time of product *i* at workstation *k*

The effective processing-time variability at workstation *k* is:

```
C_{S,k}² =
[ Σ_{i=1}^m (λ_{ik}/λ_k) · E[TS_{ik}]² · (1 + C_{S,ik}²) ]
/ E[TS_k]² − 1
```

This formula is derived using **properties of mixture distributions**.

---

## 6.1.7 Composite Routing Matrix

Each product type has its own routing matrix.
A **composite routing matrix** is required.

### Composite Routing Probability
```
p_{jk} =
[ Σ_{i=1}^m λ_{ij} · p_{jk}^{(i)} ] / λ_j
```

Where:
- `p_{jk}^{(i)}` = routing probability for product *i*
- `λ_j` = total arrival rate at workstation *j*

This matrix is used for:
- arrival-rate computation,
- arrival-variability computation.

---

## 6.1.8 Arrival Variability
Arrival variability is computed using the **general network approximation model**:

1. Composite routing matrix  
2. Departure variability:
```
C_{D,k}² ≈ (1 − U_k²)·C_{A,k}² + U_k²·C_{S,k}²
```
3. Arrival variability equations:
```
C_{A,k}² = merging of external and internal flows
```

This yields a system of equations in `C_{A,k}²`.

---

## 6.1.9 Performance of Each Workstation

Queue cycle time (VUT):
```
CTQ_k ≈ (C_{A,k}² + C_{S,k}²)/2 · U_k/(1 − U_k) · E[TS_k]
```

Total cycle time:
```
CT_k = CTQ_k + E[TS_k]
```

Work in process:
```
WIP_k = λ_k · CT_k
```

---

## 6.1.10 Network Performance

- Total WIP:
```
WIP_total = Σ_{k=1}^n WIP_k
```

- Throughput:
```
TH = Σ_i γ_i
```

- Total cycle time:
```
CT_total = WIP_total / TH
```

---

## 6.1.11 Cycle Time by Product Type

Average cycle time of product *i*:
```
CT_i = WIP_i / γ_i
```

Where:
```
WIP_i = Σ_{k=1}^n λ_{ik} · CT_{ik}
CT_{ik} = CTQ_k + E[TS_{ik}]
```

---

## 6.1.12 Key Takeaways
- Multiple-product systems extend network models.
- Workload aggregates arrival rates and processing times.
- Processing times are **mixtures of distributions**.
- Composite routing matrices are essential.
- Performance can be computed:
  - per workstation,
  - per product type,
  - for the entire system.
