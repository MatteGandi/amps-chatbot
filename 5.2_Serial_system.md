# 5.2 Serial Systems – Performance Analysis

## 5.2.1 Objective
This section explains how to **compute the performance of a serial production system** (aligned workstations), and then generalizes the procedure to more complex networks.

The key idea is:
> analyze each workstation **separately**, starting from the first one, and propagate variability downstream.

---

## 5.2.2 General Procedure for Serial Systems

For a line of workstations in series:

1. Analyze **Workstation 1**
   - compute utilization
   - compute queue cycle time
   - compute cycle time and WIP
   - compute **departure variability**

2. Use the departure parameters of WS1 as **arrival parameters** for WS2

3. Repeat for WS2, WS3, …

4. Compute system-level performance:
   - Total WIP = sum of WIP of all workstations
   - Total Cycle Time = Total WIP / Throughput (Little’s Law)

This procedure is **always valid**, even for non-exponential systems.

---

## 5.2.3 Key Rules

- Throughput of an unlimited serial line:
```
TH = λ
```
- Total WIP:
```
WIP_system = Σ WIP_i
```
- Total cycle time:
```
CT_system = WIP_system / TH
```

Summing individual cycle times is valid **only for pure serial lines**, not for general networks.

---

## 5.2.4 Example 1 – Emergency Room (M/M/1 – M/M/1 – M/M/2)

### System Description
Patients arrive at an emergency room according to a Poisson process:
```
λ = 4 patients/hour
```
Three workstations in series:

| Workstation | Servers | Avg. Proc. Time | Distribution |
|------------|--------|-----------------|--------------|
| WS1 (Clerk) | 1 | 4 min | Exponential |
| WS2 (Doctor) | 1 | 10 min | Exponential |
| WS3 (Doctors) | 2 | 24 min | Exponential |

---

### Workstation 1 (M/M/1)

- Average inter-arrival time:
```
E[TA] = 15 min
```
- Utilization:
```
U = 4 / 15 ≈ 0.27
```
- WIP:
```
WIP ≈ 0.36 patients
```
- Departure variability:
```
C_D² = 1
```

---

### Workstation 2 (M/M/1)

- Arrival rate unchanged:
```
λ = 4 patients/hour
```
- Utilization:
```
U = 10 / 15 ≈ 0.67
```
- WIP:
```
WIP ≈ 2 patients
```
- Departure variability:
```
C_D² = 1
```

---

### Workstation 3 (M/M/2)

- Utilization:
```
U = 24 / (2·15) = 0.8
```
- Queue cycle time (M/M/c):
```
CTQ ≈ 42.7 min
```
- Total cycle time:
```
CT ≈ 66.7 min = 1.11 hours
```
- WIP:
```
WIP ≈ 4.44 patients
```

---

### System Performance

Total WIP:
```
WIP = 0.36 + 2 + 4.44 ≈ 6.8 patients
```

Total cycle time:
```
CT = 6.8 / 4 ≈ 1.7 hours
```

## 5.2.5 Example 2 – Serial G/G/1 Line

### Given
- Arrival process:
```
λ = 4 jobs/hour
C_A² = 0.75
```
- Processing parameters:

| WS | E[TS] (min) | C_S² |
|----|------------|------|
| 1 | 12 | 2 |
| 2 | 9 | 0.7 |
| 3 | 13.2 | 1 |

---

### Workstation 1 (G/G/1)

- Utilization:
```
U = 12 / 15 = 0.8
```
- Cycle time (VUT):
```
CT ≈ 78 min
```
- WIP:
```
WIP ≈ 5.2 jobs
```
- Departure variability:
```
C_D² ≈ 1.55
```

---

### Workstation 2 (G/G/1)

- Arrival variability:
```
C_A² = 1.55
```
- Utilization:
```
U = 9 / 15 = 0.6
```
- Cycle time:
```
CT ≈ 0.4 hours
```
- WIP:
```
WIP ≈ 1.6 jobs
```
- Departure variability:
```
C_D² ≈ 1.24
```

---

### Workstation 3 (G/G/1)

- Arrival variability:
```
C_A² = 1.24
```

- Utilization:

```
U = 13.2 / 15 = 0.88
```
- Cycle time:
```
CT ≈ 2.03 hours
```

```
WIP ≈ 8.12 jobs
```

Total WIP:

```
WIP ≈ 14.9 jobs
```

Total cycle time:


---

## 5.2.6 Ordering Workstations – Variability Rule

When choosing the order of workstations:

> **Place the workstation with lower variability first.**

Reason:
- variability propagates downstream,
- early variability affects all subsequent stations.

This rule allows correct decisions **without full computations**.

---

## 5.2.7 Key Takeaways

- Serial systems are analyzed one workstation at a time.
- Departure variability becomes arrival variability downstream.
- Total WIP is always additive.
- Variability propagation dominates performance.
- Ordering low-variability stations first improves system performance.
