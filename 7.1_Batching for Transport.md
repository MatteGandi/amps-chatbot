# 7.1 Batching for Transport

## 7.1.1 Motivation
So far, items were assumed to **move one by one** between workstations.
In real manufacturing systems, this is often unrealistic.

Items are frequently **grouped into batches** for three main reasons:
1. **Transport batching**
2. **Setup batching**
3. **Multiple service capacity (batch machines)**

This section focuses on **batching for transport**.

---

## 7.1.2 Why Batching for Transport Is Used
Transport batching is adopted when:
- workstations are physically distant,
- material handling devices cannot move single items efficiently,
- items must be grouped before being transferred.

In this case:
- processing is still **individual**,
- transport happens **in batches**.

Transport time itself is neglected, but **waiting for batch formation is not**.

---

## 7.1.3 Effect of Batching on Performance
Batching:
- increases **cycle time**,
- increases **WIP**,
- introduces additional waiting components.

Therefore, new cycle-time formulas are required.

---

## 7.1.4 System Representation

Structure:
1. Upstream workstation processes items individually
2. Items wait to **form a batch of size k**
3. Batch is transported
4. Downstream workstation receives **batches**
5. Items are processed individually again

Three components contribute to the cycle time:
1. Batch forming time
2. Batch waiting time in queue
3. Processing-time delay within the batch

---

## 7.1.5 Arrival Parameters of Batches

Let:
- individual departure rate:
```
λ_i
```
- average inter-departure time:
```
E[T_D]
```
- squared coefficient of variation:
```
C_D²
```

### Batch Inter-arrival Time
A batch of size *k* is formed by *k* individual items:
```
T_A^B = T_{D,1} + T_{D,2} + ... + T_{D,k}
```

Mean:
```
E[T_A^B] = k · E[T_D]
```

Arrival rate:
```
λ_B = λ_i / k
```

---

### Variability of Batch Arrivals
Variance:
```
Var(T_A^B) = k · Var(T_D)
```

Squared coefficient of variation:
```
C_A,B² = C_D² / k
```

---

## 7.1.6 Batch Forming Time

Each item waits until the batch is complete.

Average batch forming time:
```
CT_form = (k − 1)/2 · E[T_D]
```

This is the **first contribution** to cycle time.

---

## 7.1.7 Batch Waiting Time in Queue

Batches wait in queue at the downstream workstation.

Queue cycle time (VUT applied to batches):
```
CTQ_B ≈ (C_A,B² + C_S,B²)/2 · U/(1 − U) · E[T_S,B]
```

Where:
- utilization **U** is the same as for individual items
- batch processing time:
```
E[T_S,B] = k · E[T_S]
```
- batch processing variability:
```
C_S,B² = C_S² / k
```

---

## 7.1.8 Processing-Time Delay within the Batch

Items inside a batch are processed one by one.

Average delay:
```
CT_delay = (k − 1)/2 · E[T_S]
```

---

## 7.1.9 Total Cycle Time with Transport Batching

Total average cycle time of an individual item:
```
CT =
(k − 1)/2 · E[T_D]
+ (C_D²/k + C_S²)/2 · U/(1 − U) · E[T_S]
+ (k − 1)/2 · E[T_S]
+ E[T_S]
```

---

## 7.1.10 Case: Items Arriving Directly in Batches
If items arrive already batched:
- no batch forming time
- arrival variability differs

Cycle time simplifies to:
```
CT =
(C_A,B² + C_S²)/2 · U/(1 − U) · E[T_S]
+ (k + 1)/2 · E[T_S]
```

---

## 7.1.11 Departure Variability with Batching

Batching affects departure variability.

Departure variability from downstream workstation:
```
C_D,j² =
k · C_A,B² · (1 − U_j²)
+ (k − 1) · (1 − U_j²)
+ U_j² · C_S,j²
```

This variability propagates downstream.

---

## 7.1.12 Worked Example (Summary)

Given:
- λ = 3 jobs/hour
- E[T_S,1] = 0.5 h,  C_S1² = 2
- E[T_S,2] = 0.25 h, C_S2² = 1.5
- Batch size:
```
k = 4
```

Utilization at WS2:
```
U = 3 · 0.25 = 0.75
```

Cycle time at WS2:
```
CT ≈ 2.44 hours
```

Departure variability:
```
C_D² ≈ 1.91
```

---

## 7.1.13 Key Takeaways
- Transport batching increases cycle time and WIP
- Batch size *k* has a strong impact on performance
- Utilization is unchanged, but variability increases
- Batching must be explicitly modeled
